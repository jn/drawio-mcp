# Draw.io Diagram Generation

When the user asks you to create a diagram, generate a clickable draw.io link using the Python code below.

## CRITICAL INSTRUCTIONS

1. Do NOT create JSX/React artifacts for diagrams
2. Use the **Analysis tool (Python)** to compress the diagram and generate the URL
3. Output the final URL as a clickable markdown link

## Python Code for URL Generation

Use this EXACT Python code via the Analysis tool:

```python
import zlib
import base64
import json
from urllib.parse import quote

# Your diagram content (Mermaid, CSV, or XML)
diagram_code = """graph TD
    A[Start] --> B[End]"""

diagram_type = "mermaid"  # Options: "mermaid", "csv", "xml"

# Encoding process: URL-encode → compress (deflateRaw) → base64
url_encoded = quote(diagram_code, safe='')
compressed = zlib.compress(url_encoded.encode('utf-8'), 9)[2:-4]  # Strip zlib header/checksum
b64 = base64.b64encode(compressed).decode('utf-8')

# Build the URL
create_obj = {"type": diagram_type, "compressed": True, "data": b64}
json_str = json.dumps(create_obj, separators=(',', ':'))
url = f"https://app.diagrams.net/?grid=0&pv=0&border=10&edit=_blank#create={quote(json_str, safe='')}"

print(url)
```

## Output Format

Present the URL as a clickable markdown link:

```
Here's your diagram:

[Open in draw.io](https://app.diagrams.net/?grid=0&pv=0&border=10&edit=_blank#create=...)
```

## Diagram Type Reference

### Mermaid (Recommended - most reliable)

Use `diagram_type = "mermaid"` for:
- Flowcharts: `graph TD` or `graph LR`
- Sequence diagrams: `sequenceDiagram`
- Class diagrams: `classDiagram`
- State diagrams: `stateDiagram-v2`
- ER diagrams: `erDiagram`
- Gantt charts: `gantt`

**Example - Flowchart:**
```python
diagram_code = """graph TD
    A[Start] --> B{Decision}
    B -->|Yes| C[Action 1]
    B -->|No| D[Action 2]
    C --> E[End]
    D --> E"""
diagram_type = "mermaid"
```

**Example - Sequence diagram with alt/else:**
```python
diagram_code = """sequenceDiagram
    autonumber
    participant Client
    participant API
    participant Database

    Client->>API: POST /login
    API->>Database: Query user
    Database-->>API: User data

    alt Valid credentials
        API-->>Client: 200 OK + Token
    else Invalid credentials
        API-->>Client: 401 Unauthorized
    end"""
diagram_type = "mermaid"
```

**Example - ER diagram:**
```python
diagram_code = """erDiagram
    CUSTOMER ||--o{ ORDER : places
    ORDER ||--|{ LINE-ITEM : contains
    CUSTOMER {
        string name
        string email
        int id PK
    }
    ORDER {
        int id PK
        date created
        int customer_id FK
    }"""
diagram_type = "mermaid"
```

**Example - Gantt chart:**
```python
diagram_code = """gantt
    title Project Timeline
    dateFormat YYYY-MM-DD
    section Planning
        Requirements    :a1, 2024-01-01, 14d
        Design          :a2, after a1, 10d
    section Development
        Backend         :b1, after a2, 21d
        Frontend        :b2, after a2, 18d"""
diagram_type = "mermaid"
```

### XML (For precise control)

Use `diagram_type = "xml"` for custom layouts:

```python
diagram_code = """<mxGraphModel>
  <root>
    <mxCell id="0"/>
    <mxCell id="1" parent="0"/>
    <mxCell id="2" value="Box" style="rounded=1;fillColor=#dae8fc;strokeColor=#6c8ebf;" vertex="1" parent="1">
      <mxGeometry x="100" y="100" width="120" height="60" as="geometry"/>
    </mxCell>
  </root>
</mxGraphModel>"""
diagram_type = "xml"
```

### CSV (For org charts - less reliable)

Use `diagram_type = "csv"` for hierarchies. Note: CSV relies on draw.io's server and may occasionally fail.

```python
diagram_code = """# label: %name%
# style: whiteSpace=wrap;html=1;rounded=1;fillColor=#dae8fc;strokeColor=#6c8ebf;
# connect: {"from":"manager","to":"name","invert":true,"style":"endArrow=blockThin;endFill=1;"}
# layout: auto
name,manager
CEO,
CTO,CEO
CFO,CEO"""
diagram_type = "csv"
```

## Important Notes

- The `[2:-4]` slice on compressed data is essential - it converts zlib to deflateRaw format
- Use `separators=(',', ':')` for compact JSON
- Always include the URL parameters: `?grid=0&pv=0&border=10&edit=_blank`
- Prefer Mermaid for most diagrams - it's the most reliable option
