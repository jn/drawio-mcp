# Draw.io Diagram Generation

When the user asks you to create a diagram, generate a clickable draw.io link.

## CRITICAL INSTRUCTIONS

1. Do NOT create JSX/React artifacts
2. Use the **Analysis tool (Python)** to compress the diagram and generate the URL
3. Output the final URL as a clickable markdown link in your response

## How to Generate the URL

Use Python (via the Analysis tool) with this EXACT code:

```python
import zlib
import base64
import json
from urllib.parse import quote

# Your diagram code (Mermaid, CSV, or XML)
diagram_code = """graph TD
    A[Start] --> B[End]"""

diagram_type = "mermaid"  # or "csv" or "xml"

# IMPORTANT: First URL-encode, then compress, then base64
# Step 1: URL encode the diagram code
url_encoded = quote(diagram_code, safe='')

# Step 2: Compress with deflateRaw (zlib without header/checksum)
compressed = zlib.compress(url_encoded.encode('utf-8'), 9)[2:-4]

# Step 3: Base64 encode the compressed data
b64 = base64.b64encode(compressed).decode('utf-8')

# Use #create hash with JSON wrapper
create_obj = {"type": diagram_type, "compressed": True, "data": b64}
json_str = json.dumps(create_obj, separators=(',', ':'))  # compact JSON
url = f"https://app.diagrams.net/#create={quote(json_str, safe='')}"

print(url)
```

## Output Format

After running the Python script, present the URL as a clickable link:

```
Here's your diagram:

👉 [Open in draw.io](https://app.diagrams.net/?pv=0&grid=0#create=...)
```

## Diagram Types

- `"mermaid"` - Flowcharts, sequence diagrams, class diagrams, state diagrams, ER diagrams, Gantt charts
- `"csv"` - Org charts, hierarchical data (use draw.io CSV format with # directives)
- `"xml"` - Native draw.io XML

## Important

- ALWAYS use the Analysis/Python tool - never try to do encoding manually
- NEVER create JSX or React artifacts
- The encoding order is critical: URL-encode → compress → base64
- Use `separators=(',', ':')` in json.dumps for compact output
- Present the final link directly in your response as markdown
